# PayBill Engineering Blogs

Welcome to the engineering documentation and technical blog series for **PayBill**, a modern React Native application designed for seamless, reliable, and secure financial transactions.

This series dives deep into the architecture, design patterns, and engineering decisions that power the application across mobile and web platforms.

## Table of Contents

1. **[Building PayBill (Main Overview)](./paybill-engineering-blog.md)**
   A high-level overview of the problem statement, system architecture, technology stack, and the engineering decisions behind the UX.

2. **[Architecture: Robust Role-Based Access Control and Secure Authentication](./paybill-security-auth-rbac-blog.md)**
   An exploration of how we implemented zero-trust API boundaries, encrypted biometric session locks, and rapid multi-user account switching.

3. **[Forms: Validation Binding and the Approval Engine](./paybill-approval-forms-blog.md)**
   How we divorced UI from business logic using Zod and React Hook Form, and how our proprietary state-machine powers complex financial approval workflows.

4. **[Seamless QR Code Scanner and Generator](./paybill-qr-scanning-blog.md)**
   The engineering behind an "always-active" hardware-accelerated QR scanner and dynamic bottom-sheet receipt generation.

5. **[Offline First: Auto Re-Sync and Edge Reliability Architecture](./paybill-offline-sync-blog.md)**
   How PayBill handles network drops gracefully using local storage drafts and persistent background queues for automatic data reconciliation.
